# OpenShift Agent-Based Installer - Deployment Guide (llm.txt)

## Overview

### Project Description
OpenShift Agent Based Installer Helper - Utilities to easily leverage the OpenShift Agent-Based Installer for automated cluster deployments.

**Repository**: https://github.com/tosin2013/openshift-agent-install
**License**: Apache-2.0
**Supported OpenShift Versions**: 4.15+, tested with 4.20.x and 4.21.x

### Deployment Capabilities
- **Platforms**: Bare metal, vSphere, platform=none
- **Patterns**: Single Node OpenShift (SNO), 3-node compact clusters, HA clusters (3 control + N workers)
- **Infrastructure**: Declarative YAML-based configuration
- **DNS**: Automated dnsmasq-based DNS setup (replaces FreeIPA dependency)
- **Testing**: E2E testing with KVM/libvirt automation

### Key Features
- Declarative cluster configuration via `cluster.yml` and `nodes.yml`
- Automated ISO generation with Ansible templating
- KVM/libvirt deployment automation with Redfish emulation
- Automated DNS configuration and cleanup
- Network configuration via NMState (VLAN, Bond, Bond+VLAN support)
- Disconnected/air-gap registry support
- Appliance image creation for edge deployments

## Quick Start

### Fastest Path to Deployment

```bash
# 1. Bootstrap environment (one-time setup - installs packages, DNS, router)
sudo ./e2e-tests/bootstrap_env.sh

# 2. Configure environment variables (optional - defaults shown)
export SITE_CONFIG_DIR="examples"              # Where cluster configs live
export GENERATED_ASSET_PATH="~/generated_assets"  # Where ISOs/manifests go

# 3. Download OpenShift CLI tools
./download-openshift-cli.sh
sudo cp ./bin/* /usr/local/bin/

# 4. Create pull secret file
cat > ~/ocp-install-pull-secret.json << 'EOF'
{your-pull-secret-json-here}
EOF

# 5. Setup DNS server (automated in bootstrap, or run manually)
sudo ./hack/setup-dnsmasq.sh

# 6. Configure DNS entries for your cluster
sudo ./hack/configure-dnsmasq-entries.sh add examples/sno-4.20-standard/cluster.yml

# 7. Create cluster ISO
./hack/create-iso.sh sno-4.20-standard

# 8. Deploy to KVM (for testing)
./hack/deploy-on-kvm.sh examples/sno-4.20-standard/nodes.yml --redfish

# 9. Watch and auto-reboot VMs (REQUIRED for KVM - run in background)
./hack/watch-and-reboot-kvm-vms.sh examples/sno-4.20-standard/nodes.yml &

# 10. Monitor installation
./bin/openshift-install agent wait-for bootstrap-complete --dir ~/generated_assets/sno-4-20/
./bin/openshift-install agent wait-for install-complete --dir ~/generated_assets/sno-4-20/

# 11. Access cluster
export KUBECONFIG=~/generated_assets/sno-4-20/auth/kubeconfig
oc get nodes
```

## Deployment Steps (Ordered Workflow)

### Phase 0: Environment Bootstrap (One-Time Setup)

**Purpose**: Install all dependencies, configure infrastructure, setup DNS server

**Script**: `e2e-tests/bootstrap_env.sh`

**Actions**:
- Installs system packages (ansible, nmstate, libvirt, etc.)
- Downloads and installs OpenShift CLI tools
- Installs Ansible collections
- Configures libvirtd and virtualization
- Sets up dnsmasq DNS server (replaces FreeIPA)
- Deploys VyOS router for network infrastructure
- Configures LVM for libvirt storage
- Sets up container registry authentication

**Command**:
```bash
sudo ./e2e-tests/bootstrap_env.sh
```

**Validation**:
```bash
./e2e-tests/validate_env.sh
```

---

### Phase 1: Infrastructure Preparation

**Purpose**: Configure network infrastructure and DNS

**Scripts**:
- `hack/setup-dnsmasq.sh` - Install and configure dnsmasq DNS server
- `hack/vyos-router.sh` - Deploy VyOS router VM for network infrastructure
- `hack/configure-lvm.sh` - Configure LVM storage for libvirt

**DNS Setup**:
```bash
# Install dnsmasq (if not done in bootstrap)
sudo ./hack/setup-dnsmasq.sh

# Add DNS entries for cluster
sudo ./hack/configure-dnsmasq-entries.sh add examples/sno-4.20-standard/cluster.yml

# Verify DNS resolution
dig @localhost api.sno-4-20.example.com
dig @localhost console-openshift-console.apps.sno-4-20.example.com
```

---

### Phase 2: Cluster Configuration

**Purpose**: Define cluster topology and network configuration

**Configuration Files**:
- `examples/<cluster-name>/cluster.yml` - Cluster metadata, VIPs, networking
- `examples/<cluster-name>/nodes.yml` - Node definitions, MAC addresses, NMState config

**Key Parameters** (see Configuration Reference section for full details):
- `cluster_name`: Cluster identifier
- `base_domain`: DNS domain
- `api_vips`: API endpoint VIP(s)
- `app_vips`: Application ingress VIP(s)
- `platform_type`: baremetal, vsphere, or none (SNO must be "none")
- `control_plane_replicas`: 1 (SNO), 3 (3-node/HA)
- `app_node_replicas`: 0 (SNO/3-node), N (HA)

**Example Configurations**:
- `examples/sno-4.20-standard/` - Standard SNO deployment
- `examples/serenity-sno.v60.lab.kemo.network/` - SNO with disconnected registry
- `examples/ha-4.21-disconnected/` - HA cluster with disconnected registry

---

### Phase 3: ISO Generation

**Purpose**: Generate Agent-Based Installer ISO with cluster configuration

**Scripts**:
- `hack/create-iso.sh` - Main ISO creation script
- `playbooks/create-manifests.yml` - Ansible playbook for manifest templating

**Process**:
1. Script reads `cluster.yml` and `nodes.yml` from specified example directory
2. Ansible playbook templates Agent-Based Installer manifests
3. `openshift-install agent create image` generates bootable ISO
4. ISO saved to `${GENERATED_ASSET_PATH}/${CLUSTER_NAME}/agent.x86_64.iso`

**Command**:
```bash
./hack/create-iso.sh sno-4.20-standard
```

**Output Location**:
```
~/generated_assets/sno-4-20/agent.x86_64.iso
```

---

### Phase 4: Deployment Execution

#### Option A: KVM/Libvirt Deployment (Testing)

**Purpose**: Deploy cluster to local KVM hypervisor with automated VM creation

**Script**: `hack/deploy-on-kvm.sh`

**Features**:
- Automatic VM resource allocation based on node count
- Redfish API emulation via sushy-tools
- Automated DNS configuration in libvirt network
- Multi-network support (dual NICs)
- ODF storage provisioning (optional)

**Command**:
```bash
./hack/deploy-on-kvm.sh examples/sno-4.20-standard/nodes.yml --redfish
```

**What It Does**:
1. Copies ISO to `/var/lib/libvirt/images/`
2. Configures DNS entries in libvirt network
3. Configures host to use libvirt DNS
4. Creates VMs with specifications from `nodes.yml`
5. Attaches ISO and boots VMs
6. Starts sushy-tools Redfish emulator (if --redfish flag)

**IMPORTANT**: After deploying, immediately start the VM reboot watcher:
```bash
./hack/watch-and-reboot-kvm-vms.sh examples/sno-4.20-standard/nodes.yml &
```
This is **required** for Agent-Based Installer - VMs will shut down after writing OS to disk but won't automatically reboot without this script.

#### Option B: Physical/vSphere Deployment

**Purpose**: Deploy to bare metal or vSphere infrastructure

**Process**:
1. Copy generated ISO to accessible location
2. Attach ISO to physical servers or vSphere VMs
3. Boot servers from ISO
4. Installation proceeds automatically

**ISO Location**: `~/generated_assets/${CLUSTER_NAME}/agent.x86_64.iso`

---

### Phase 5: Installation Monitoring

**Purpose**: Monitor cluster bootstrap and installation progress

**Commands**:
```bash
# Watch bootstrap process
./bin/openshift-install agent wait-for bootstrap-complete \
    --dir ~/generated_assets/${CLUSTER_NAME}/

# Watch full installation
./bin/openshift-install agent wait-for install-complete \
    --dir ~/generated_assets/${CLUSTER_NAME}/
```

**Required for KVM Deployments**:
```bash
# Watch and auto-reboot VMs after disk write (REQUIRED for KVM)
# Agent-Based Installer VMs shut down after writing OS to disk but don't self-reboot
# This script watches for shutdowns and automatically restarts them
./hack/watch-and-reboot-kvm-vms.sh examples/sno-4.20-standard/nodes.yml

# Recommended: Run in background
./hack/watch-and-reboot-kvm-vms.sh examples/sno-4.20-standard/nodes.yml &
```

---

### Phase 6: Cluster Access

**Purpose**: Access deployed cluster via CLI and Web Console

**CLI Access**:
```bash
export KUBECONFIG=~/generated_assets/${CLUSTER_NAME}/auth/kubeconfig
oc get nodes
oc get co
```

**Web Console Access**:
```
URL: https://console-openshift-console.apps.${CLUSTER_NAME}.${BASE_DOMAIN}/
Username: kubeadmin
Password: cat ~/generated_assets/${CLUSTER_NAME}/auth/kubeadmin-password
```

---

### Phase 7: Cleanup (Destruction)

**Purpose**: Remove cluster and clean up resources

**Script**: `hack/destroy-on-kvm.sh`

**Command**:
```bash
./hack/destroy-on-kvm.sh examples/sno-4.20-standard/nodes.yml
```

**Actions**:
1. Removes DNS entries from libvirt network
2. Destroys VMs
3. Deletes VM disk images
4. Removes ODF storage disks (if present)
5. Deletes agent ISO

**Manual DNS Cleanup** (if needed):
```bash
sudo ./hack/configure-dnsmasq-entries.sh remove sno-4-20 example.com
```

- `api.${cluster_name}.${base_domain}` â†’ API VIP
- `api-int.${cluster_name}.${base_domain}` â†’ API VIP
- `*.apps.${cluster_name}.${base_domain}` â†’ App VIP (wildcard)

**Example**:
```bash
# Add DNS entries for cluster
sudo ./hack/configure-dnsmasq-entries.sh add examples/sno-4.20-standard/cluster.yml

# Or use shortcut
sudo ./hack/configure-dnsmasq-entries.sh examples/sno-4.20-standard/cluster.yml
```

### deploy-on-kvm.sh

**Location**: `/home/vpcuser/openshift-agent-install/hack/deploy-on-kvm.sh`

**Description**: Deploys OpenShift cluster to KVM/libvirt infrastructure. Creates VMs, configures DNS, and starts installation.

**Usage**:
```bash
./hack/deploy-on-kvm.sh <nodes_yaml_file> [--redfish]
```

**Arguments**:
- `<nodes_yaml_file>` - Path to nodes.yml configuration file
- `--redfish` - (Optional) Enable Redfish API emulation via sushy-tools

**Environment Variables**:
- `CLUSTER_NAME` - Cluster name (auto-detected from cluster.yml, default: "ocp4")
- `GENERATED_ASSET_PATH` - Path to generated assets (default: "~/generated_assets")
- `LIBVIRT_NETWORK` - Libvirt network for VMs (default: "network=1924,model=e1000e")
- `LIBVIRT_NETWORK_TWO` - Second network for multi-NIC (default: same as LIBVIRT_NETWORK)
- `LIBVIRT_VM_PATH` - Path to VM disks (default: "/var/lib/libvirt/images")
- `MULTI_NETWORK` - Enable dual NICs (default: true)
- `SUSHY_TOOLS_IMAGE` - Sushy-tools container image (default: quay.io/metal3-io/sushy-tools)

**Dependencies**:
- libvirt/KVM
- virt-install
- yq
- podman (if using --redfish)
- Agent ISO must exist at ${GENERATED_ASSET_PATH}/${CLUSTER_NAME}/agent.x86_64.iso

**Workflow Phase**: Phase 4 (Deployment Execution)

**Resource Allocation** (automatic based on node count):
- **SNO (1 node)**: 8 vCPU, 32GB RAM, 130GB disk + 400GB ODF disk
- **3-node**: 6 vCPU per node, 32GB RAM, 130GB disk + 400GB ODF disk
- **5-node**: Control (6 vCPU, 24GB RAM), Workers (6 vCPU, 24GB RAM)
- **6+ node**: Control (6 vCPU, 24GB RAM), Workers (12 vCPU, 32-48GB RAM)

**Features**:
- Automatic VM creation from nodes.yml
- MAC address assignment per node
- DNS configuration in libvirt network
- Host DNS configuration to use libvirt
- Redfish API emulation (optional)
- Multi-network support
- ODF storage provisioning

**Example**:
```bash
# Deploy with Redfish enabled
./hack/deploy-on-kvm.sh examples/sno-4.20-standard/nodes.yml --redfish

# Deploy without Redfish
./hack/deploy-on-kvm.sh examples/sno-4.20-standard/nodes.yml

# Custom cluster name
export CLUSTER_NAME="my-cluster"
./hack/deploy-on-kvm.sh examples/sno-4.20-standard/nodes.yml --redfish

# Test Redfish API after deployment
curl http://localhost:8000/redfish/v1/Systems
```

### destroy-on-kvm.sh

**Location**: `/home/vpcuser/openshift-agent-install/hack/destroy-on-kvm.sh`

**Description**: Destroys OpenShift cluster on KVM. Removes VMs, disks, DNS entries, and agent ISO.

**Usage**:
```bash
./hack/destroy-on-kvm.sh <nodes_yaml_file>
```

**Arguments**:
- `<nodes_yaml_file>` - Path to nodes.yml configuration file

**Environment Variables**:
- `CLUSTER_NAME` - Cluster name (auto-detected from cluster.yml, default: "ocp4")
- `LIBVIRT_VM_PATH` - Path to VM disks (default: "/var/lib/libvirt/images")

**Dependencies**:
- virsh (libvirt CLI)
- yq
- Root/sudo access

**Workflow Phase**: Phase 7 (Cleanup)

**Actions**:
1. Removes DNS entries from libvirt network
2. Destroys all VMs defined in nodes.yml
3. Deletes VM disk images (${CLUSTER_NAME}-${node_name}.qcow2)
4. Deletes ODF storage disks (${CLUSTER_NAME}-${node_name}-odf.qcow2)
5. Removes agent ISO (/var/lib/libvirt/images/agent.x86_64.iso)

**Example**:
```bash
# Destroy cluster
./hack/destroy-on-kvm.sh examples/sno-4.20-standard/nodes.yml

# With explicit cluster name
export CLUSTER_NAME="my-cluster"
./hack/destroy-on-kvm.sh examples/sno-4.20-standard/nodes.yml

# Verify cleanup
sudo virsh list --all
sudo ls -lh /var/lib/libvirt/images/
```

### watch-and-reboot-kvm-vms.sh

**Location**: `/home/vpcuser/openshift-agent-install/hack/watch-and-reboot-kvm-vms.sh`

**Description**: **REQUIRED for KVM deployments** - Watches for VMs to shut down after Agent-Based Installer writes RHCOS to disk, then automatically reboots them. VMs do not self-reboot after disk write; without this script, installation will hang.

**Usage**:
```bash
./hack/watch-and-reboot-kvm-vms.sh <nodes_yaml_file>
```

**Arguments**:
- `<nodes_yaml_file>` - Path to nodes.yml configuration file

**Environment Variables**: None

**Dependencies**:
- virsh (libvirt CLI)
- yq
- Running VMs

**Workflow Phase**: Phase 5 (Installation Monitoring) - Start immediately after deploy-on-kvm.sh

**How It Works**:
- Continuously polls `virsh list --state-shutoff` for powered-off VMs
- When a cluster VM shuts down (after writing OS to disk), automatically runs `virsh start <vm>`
- Removes VM from watch list after successful restart
- Exits when all VMs have been restarted once
- Checks every 30 seconds

**Example**:
```bash
# Monitor and auto-reboot VMs during installation
./hack/watch-and-reboot-kvm-vms.sh examples/sno-4.20-standard/nodes.yml

# Run in background
./hack/watch-and-reboot-kvm-vms.sh examples/sno-4.20-standard/nodes.yml &

# Monitor with logging
./hack/watch-and-reboot-kvm-vms.sh examples/sno-4.20-standard/nodes.yml 2>&1 | tee vm-monitor.log
```

## Configuration Reference

### cluster.yml Parameters (Alphabetically Sorted)

**Location**: `examples/<cluster-name>/cluster.yml`

**Parameters**:

- `additional_trust_bundle` (string, optional) - Additional CA certificates to trust
- `additional_trust_bundle_policy` (string, optional) - When to apply trust bundle ("Always", "Proxyonly")
- `api_vips` (list, required) - API endpoint VIP addresses (list for dual-stack)
- `app_node_replicas` (integer, required) - Number of worker nodes (0 for SNO/compact, N for HA)
- `app_vips` (list, required) - Application ingress VIP addresses (list for dual-stack)
- `base_domain` (string, required) - DNS base domain for cluster
- `cluster_name` (string, required) - Cluster identifier (used in FQDN)
- `cluster_network_cidr` (string, required) - CIDR for pod network (default: 10.128.0.0/14)
- `cluster_network_host_prefix` (integer, required) - Subnet prefix per node (default: 23)
- `control_plane_replicas` (integer, required) - Number of control plane nodes (1 for SNO, 3 for HA)
- `disconnected_registries` (list, optional) - Mirror registry configuration for air-gap
  - `source` (string) - Original registry
  - `target` (string) - Mirror registry
- `dns_search_domains` (list, optional) - DNS search domains
- `dns_servers` (list, optional) - DNS server IP addresses
- `machine_network_cidrs` (list, required) - CIDR for node network
- `network_type` (string, required) - SDN type ("OVNKubernetes" or "OpenShiftSDN" - deprecated)
- `ntp_servers` (list, optional) - NTP server addresses
- `platform_type` (string, required) - Platform type ("baremetal", "vsphere", "none" - SNO must be "none")
- `proxy` (object, optional) - Outbound proxy configuration
  - `http_proxy` (string) - HTTP proxy URL
  - `https_proxy` (string) - HTTPS proxy URL
  - `no_proxy` (list) - Proxy exclusions
- `pull_secret_path` (string, required) - Path to OpenShift pull secret file
- `rendezvous_ip` (string, required) - IP of bootstrap node (matches SNO node IP or first control plane)
- `service_network_cidrs` (list, required) - CIDR for service network (default: 172.30.0.0/16)
- `ssh_public_key_path` (string, optional) - Path to SSH public key (generated if omitted)
- `use_site_configs` (boolean, optional) - Use site-specific configurations (default: false)

**Example cluster.yml** (SNO):
```yaml

  - 192.168.100.1

  - 172.30.0.0/16
```

### nodes.yml Parameters (Alphabetically Sorted)

**Location**: `examples/<cluster-name>/nodes.yml`

**Parameters**:

- `app_node_replicas` (integer, required) - Number of worker nodes
- `control_plane_replicas` (integer, required) - Number of control plane nodes
- `nodes` (list, required) - List of node definitions
  - `hostname` (string, required) - Node hostname
  - `interfaces` (list, required) - Network interface definitions
    - `mac_address` (string, required) - MAC address
    - `name` (string, required) - Interface name (e.g., enp1s0, ens192)
  - `networkConfig` (object, required) - NMState network configuration
    - `interfaces` (list) - Interface configurations
      - `ipv4` (object) - IPv4 configuration
        - `address` (list) - Static IP addresses
          - `ip` (string) - IP address
          - `prefix-length` (integer) - Subnet prefix
        - `dhcp` (boolean) - Enable DHCP
        - `enabled` (boolean) - Enable IPv4
      - `link-aggregation` (object, optional) - Bond configuration
        - `mode` (string) - Bond mode (802.3ad, active-backup, etc.)
        - `port` (list) - Member interfaces
      - `mac-address` (string) - MAC address
      - `name` (string) - Interface name
      - `state` (string) - Interface state ("up", "down")
      - `type` (string) - Interface type ("ethernet", "vlan", "bond")
      - `vlan` (object, optional) - VLAN configuration
        - `base-iface` (string) - Parent interface
        - `id` (integer) - VLAN ID
    - `routes` (object) - Routing configuration
      - `config` (list) - Route definitions
        - `destination` (string) - Destination CIDR
        - `next-hop-address` (string) - Gateway IP
        - `next-hop-interface` (string) - Egress interface
        - `table-id` (integer) - Routing table ID (254 = main)
  - `role` (string, optional) - Node role ("master", "worker" - omit for SNO)
  - `rootDeviceHints` (object, required) - Root disk selection
    - `deviceName` (string) - Device path (e.g., /dev/sda, /dev/nvme0n1)

**Example nodes.yml** (SNO with VLAN):
```yaml
control_plane_replicas: 1
app_node_replicas: 0

nodes:
  - hostname: sno-420
    rootDeviceHints:
      deviceName: /dev/sda
    interfaces:
      - name: ens192
        mac_address: "00:50:56:9a:12:34"
    networkConfig:
      interfaces:
        - name: ens192.100
          type: vlan
          state: up
          vlan:
            id: 100
            base-iface: ens192
          ipv4:
            enabled: true
            address:
              - ip: 192.168.100.50
                prefix-length: 24
            dhcp: false
        - name: ens192
          type: ethernet
          state: up
          mac-address: "00:50:56:9a:12:34"
      routes:
        config:
          - destination: 0.0.0.0/0
            next-hop-address: 192.168.100.1
            next-hop-interface: ens192.100
            table-id: 254
```

## Troubleshooting

### DNS Resolution Issues

#### Problem: Cannot resolve cluster API or apps domains

**Symptoms**:
```bash
dig api.sno-4-20.example.com
# Returns NXDOMAIN or no answer
```

**Solution 1: Check dnsmasq status**
```bash
# Verify dnsmasq is running
sudo systemctl status dnsmasq

# Restart if needed
sudo systemctl restart dnsmasq

# Check logs
sudo tail -f /var/log/dnsmasq.log
```

**Solution 2: Verify DNS entries**
```bash
# List current DNS entries
sudo ./hack/configure-dnsmasq-entries.sh list

# Re-add entries if missing
sudo ./hack/configure-dnsmasq-entries.sh add examples/sno-4.20-standard/cluster.yml

# Test resolution
dig @localhost api.sno-4-20.example.com
```

**Solution 3: Check host DNS configuration**
```bash
# Verify /etc/resolv.conf includes localhost
cat /etc/resolv.conf | grep nameserver

# If 127.0.0.1 not present, check NetworkManager
nmcli connection show

# Add dnsmasq to primary connection
PRIMARY_CONN=$(nmcli -t -f NAME,DEVICE connection show --active | grep -v "lo\|virbr" | head -1 | cut -d: -f1)
sudo nmcli connection modify "$PRIMARY_CONN" ipv4.dns "127.0.0.1"
sudo nmcli connection up "$PRIMARY_CONN"
```

### KVM/Libvirt Issues

#### Problem: Cannot create VMs or VMs fail to start

**Symptoms**:
```bash
Error creating domain: internal error: process exited while connecting to monitor
```

**Solution 1: Check libvirtd**
```bash
# Verify libvirtd is running
sudo systemctl status libvirtd

# Restart libvirtd
sudo systemctl restart libvirtd

# Check for errors
sudo journalctl -u libvirtd -n 50
```

**Solution 2: Verify disk space**
```bash
# Check available space in /var/lib/libvirt/images
df -h /var/lib/libvirt/images

# Clean up old VM disks
sudo ls -lh /var/lib/libvirt/images/
sudo rm /var/lib/libvirt/images/old-cluster-*.qcow2
```

**Solution 3: Check permissions**
```bash
# Verify user is in libvirt group
groups $USER

# Add user to libvirt group if missing
sudo usermod -aG libvirt $USER

# Logout and login for group changes to take effect
```

```bash
# Check MAC addresses in nodes.yml match VM configuration
yq eval '.nodes[].interfaces[].mac_address' examples/sno-4.20-standard/nodes.yml
```

```bash
# Test NMState config with nmstatectl
yq eval '.nodes[0].networkConfig' examples/sno-4.20-standard/nodes.yml > /tmp/nmstate-test.yaml
nmstatectl gc /tmp/nmstate-test.yaml

# Check for validation errors
```

```bash
# For VLAN: Ensure base interface is up before VLAN
# For Bond: Ensure all member interfaces exist

# Verify interface names match actual hardware
# Common interface names: ens192, ens224, enp1s0, enp2s0
```

### Disconnected/Air-Gap Issues

#### Problem: Cannot pull images from disconnected registry

**Symptoms**:
```
Error: ImagePullBackOff
Failed to pull image: x509: certificate signed by unknown authority
```

**Solution 1: Verify additional trust bundle**
```bash
# Ensure additional_trust_bundle is set in cluster.yml
grep -A 10 "additional_trust_bundle:" examples/sno-4.20-standard/cluster.yml

# Bundle should include disconnected registry CA cert
```

**Solution 2: Check registry mirror configuration**
```bash
# Verify disconnected_registries in cluster.yml
yq eval '.disconnected_registries' examples/sno-4.20-standard/cluster.yml

# Ensure sourceâ†’target mappings are correct
# Must include:
# - quay.io/openshift-release-dev/ocp-release
# - quay.io/openshift-release-dev/ocp-v4.0-art-dev
```

**Solution 3: Validate pull secret**
```bash
# Check pull secret includes disconnected registry auth
cat ~/ocp-install-pull-secret.json | jq '.auths | keys'

# Should show both cloud.openshift.com AND disconnected registry
```
